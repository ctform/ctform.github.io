leetcode给定一个字符串，求最长子串的长度

发表于 2019-09-24 | 分类于 leetcode

给定一个字符串，请你找出其中不含有重复字符的 最长子串 的长度。

示例 1:

输入: "abcabcbb"
输出: 3 
解释: 因为无重复字符的最长子串是 "abc"，所以其长度为 3。

字符串拆分，找不重复字符最大字串长度，听起来很简单，实际写过程需要对java集合类要很熟悉，Set集合，无序不可重复，利用其来存储String使用charAt方法得到的char值。
解决思想是滑动窗口[i,j]，顺序比较一个个元素是否存在于集合中，不存在则加入[i,j+1]，更新并记录下连续不存在的最大持续次数，元素重复则将滑动窗口的左侧提至重复元素索引处并保存[index+1,j]。

继续进行优化，上面代码中对于窗口的移动是一次一位的，而且每次移动都还需要进行一次判断，能否在窗口中找到重复元素的同时获取其索引位置，一次性将窗口移动至重复元素右侧，将大大优化代码性能，当然这里Set集合就不能满足需求了，改用map集合，存储{char：index}对方式来实现

leetcode题目记录用链表存数值，两数相加求一个新链表

发表于 2019-09-24

给出两个非空的链表用来表示两个非负的整数。其中，它们各自的位数是按照 逆序 的方式存储的，并且它们的每个节点只能存储一位数字。如果，我们将这两个数相加起来，则会返回一个新的链表来表示它们的和。您可以假设除了数字0之外，这两个数都不会以0开头。

通过观察实现的节点结构可知，对于节点val值的赋予只能在创建节点时赋予，所以先对两链表的节点进行值计算，之后将得到的值用来创建结果链表的新节点

值得注意的输入可能，情况一：一表比另一表长，则对于更长的表，后续多出的节点需要继续的加入
情况二：一个链表可能为空，在处理前就需要对链表进行判断，是否为空
情况三：最后的进位，求和的最高位同样会可能出现进位，最后要考虑是否因为存在进位而需要多增加一个节点

nginx部署过程中的一些问题解决

发表于 2019-09-18

nginx部署反向代理，实现对于同一ip地址访问的不同端口转换

什么是反向代理？

  • 代理：通过客户机的配置，实现让一台服务器(代理服务器)代理客户机，客户的所有请求都交给代理服务器处理。
  • 反向代理：用一台服务器，代理真实服务器，用户访问时，不再是访问真实服务器，而是代理服务器。

nginx可以当做反向代理服务器来使用：

  • 我们需要提前在nginx中配置好反向代理的规则，不同的请求，交给不同的真实服务器处理
  • 当请求到达nginx，nginx会根据已经定义的规则进行请求的转发，从而实现路由功能

nginx反向代理原理图

  • 用户在访问过程中对于不同域名进行访问，由反向代理服务器接受，通过拦截url，根据配置文件内容将其转发给特定的服务器。
  • 在这里我错误理解为转发应该给虚拟机所在地址，而应该是服务所部署的主机地址:端口号 😓
